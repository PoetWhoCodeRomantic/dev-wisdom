# dev-wisdom
개발자의 지혜


## 단순함의 힘
```text
초보자는 복잡하게 만든다.
전문가는 단순하게 만든다.

명확한 코드는 6개월 후의 나를 위한 배려다.

기술은 변해도, 단순함을 향한 철학은 변하지 않는다.
```

## 리팩토링의 용기
```text
완벽한 첫 코드는 없다.
더 나은 코드를 향한 여정만 있을 뿐이다.

어제의 코드를 고치는 것은 부끄러움이 아니라,
성장의 증거다.

두려워 말고, 더 나은 방향으로 나아가라.
```

## 코드를 지우는 용기
```text
때로는 작성하는 것보다
삭제하는 것이 더 큰 가치를 만든다.

레거시 코드에 대한 집착은
과거에 대한 예의가 아니라,
미래에 대한 방해다.

"언젠가 쓸지도 몰라"는
가장 위험한 주석이다.

필요 없는 추상화,
사용하지 않는 함수,
읽히지 않는 주석을
과감히 지워라
```

## 예외는 예상 가능해야 한다
```text
에러를 무시하는 것은 어리석음이고,
에러를 던지기만 하는 것은 게으름이며,
에러를 예측하고 처리하는 것은 지혜다.
```

## 주석은 '왜'를 말한다
```text
코드는 '무엇'을 말하고,
주석은 '왜'를 말한다.
'무엇'을 주석으로 반복하지 마라.
```

## 간결함은 절제의 결과다
```text
많은 것을 안다고 모두 사용하지 마라.
필요한 것만 쓰는 용기가 진정한 실력이다.
과감히 지울 수 있는 용기가 좋은 코드를 만든다.
```

## 시간의 길
```text
시간은 흐르지 않는다.
우리가 흐를 뿐이다.

코드는 쌓이는 것이 아니라
익어가는 것이다.

기다림을 모르는 자는
완성을 알지 못한다.

지금 이 순간이 영원이다.
```

## 쓰기와 지우기
```text
초심자는 첫 줄 쓰기를 두려워하고,
숙련자는 첫 줄 지우기를 두려워한다.

코드는 기계가 아닌
미래의 나와의 대화다.

명확함은 친절이고,
간결함은 존중이다.

每日新 (매일신) -
매일 배우고, 매일 버리고, 매일 시작하라.
```

## 한 해의 끝에서
```text
끝이란
비움으로 다시 채워지는
원의 한 점이다.

어둠이 깊어야
빛이 스스로를 드러낸다.

마침표는 죽음이 아니라
침묵 속에서 익어가는
다음 말의 씨앗이다.

손을 펴야
새로운 것을 쥘 수 있듯
한 해를 내려놓아야
다음 해가 온다.
```
